# v11.3 — Genes/Entities Detail-Page Performance & UX Overhaul

**Date:** 2026-04-26
**Owner:** frontend
**Status:** spec — pending review

## 1. Problem

`/Genes/:symbol` (`app/src/views/pages/GeneView.vue`, 335 LoC) and `/Entities/:entity_id` (`app/src/views/pages/EntityView.vue`, 419 LoC) both fan out 8–10 API requests on mount and gate the entire page on a single `<BSpinner v-if="loading">`. The result: the user's curated SysNDD data (Associated Entities) paints **after** the slowest external lookup chain, the page feels sluggish, the layout wastes ~50 % of horizontal real estate at desktop widths, and several cards render an empty "no data" state instead of collapsing.

This spec covers the design contract for v11.3. Implementation plan(s) follow in `.planning/superpowers/plans/`.

## 2. Baseline measurements (2026-04-26)

Captured locally via Playwright over `make dev`, R API cache cleared between cold passes. See `.planning/perf/baseline-5-genes-parallel.json` and `.planning/perf/baseline-5-genes-fullnav.json` for the raw data.

### 2.1 Backend wall-time floor (all 8 endpoints fired in parallel)

| Gene   | Total bytes | Longest req (ms, warm) | Entity req (ms, warm) |
|--------|-------------|------------------------|-----------------------|
| GRIN2B | 535 KB      | 455 (entity)           | 455                   |
| SCN2A  | 858 KB      | 583 (entity)           | 583                   |
| MECP2  | 682 KB      | 473 (entity)           | 473                   |
| SHANK3 | 327 KB      | 393 (entity)           | 393                   |

**Median total wall-time floor: 473 ms.** The entity endpoint is the longest single request because it joins the cleaned ndd entity view to multiple lookups; gnomAD variants is the largest payload (up to 845 KB for SCN2A).

### 2.2 What the user feels today (full-nav, cold)

| Gene   | Header visible (ms) | Entities visible (ms) | Gap (ms) |
|--------|---------------------|-----------------------|----------|
| SCN2A  | 157                 | 689                   | 532      |
| MECP2  | 139                 | 661                   | 522      |
| SHANK3 | 127                 | 637                   | 510      |
| IFT172 | 128                 | 650                   | 522      |
| **Median** | **134**         | **656**               | **522**  |

The header band paints fast (~135 ms), but the **entities table — the curated data the user came for — lags by ~520 ms** because it is mounted only after `loadGeneInfo()` resolves and `nextTick` completes. External cards lag entities by another ~200–400 ms.

### 2.3 Code-level findings (`GeneView.vue`)

1. `<div v-if="loading">spinner</div>` gates the whole page until both gene calls return.
2. Two gene lookups (`?input_type=hgnc` and `?input_type=symbol`) fire concurrently as a fallback — even when the URL parameter is the canonical symbol, the second call is wasted.
3. `<TablesEntities>` only needs the URL `:symbol`, but is mounted only after the gene API resolves.
4. External composables fire after `nextTick`, then issue ad-hoc `axios.get` per source with no shared cache; back-nav from a sibling gene re-fetches everything.
5. Three nested fluid containers per section (`container-fluid > BContainer fluid > BRow > BCol`) accumulate ~30–40 px of stacked padding on every block, contributing to the sparse feel.
6. Sort-arrow visual noise: every sortable column shows ↑ AND ↓ at idle; the active sort is unclear.
7. Mobile (375 px): the entities table does not responsively stack — it horizontally scrolls inside a card.
8. Empty `Gene Constraint (gnomAD)` card always renders, even when its content is "No constraint data available for this gene".

## 4. Solution architecture

### 4.1 Data layer (no new dep)

- `app/src/composables/useResource.ts` — `useResource<T>(key, fetcher, opts)` returning `{ data, error, loading, isStale, refresh, abort }`. On mount: serve cached value (if any) immediately as `isStale`; revalidate in background; dedupe in-flight by `key`; abort on key change. Tracks `fetchedAt`.
- `app/src/stores/cacheStore.ts` — Pinia store wrapping a Map keyed by stable URL key with `{ value, fetchedAt, ttlMs }`. LRU bounded at 64 entries. TTL defaults: 60 s for "our" endpoints (gene/entity), 5 min for external endpoints (ClinVar, AlphaFold, MGI, RGD, UniProt).
- Per-source hooks: `useGeneRecord(symbol)`, `useGeneEntities(symbol)`, `useGeneClinVar(symbol)`, `useGeneAlphaFold(symbol)`, `useGeneUniProt(symbol)`, `useGeneMGI(symbol)`, `useGeneRGD(symbol)`. Symbols are reactive; the hook re-runs on change. The existing `useGeneExternalData` and `useModelOrganismData` are split into per-source hooks (one source = one hook).
- Drop the duplicate gene lookup: hit `?input_type=hgnc` only when the URL param matches `/^HGNC:?\d+$/`; otherwise hit `?input_type=symbol`. Saves one round trip per page hit.

### 4.2 Hydration order

1. **Entities first.** `<TablesEntities :symbol>` mounts immediately on page mount; no gating on the gene record. The hook fires the entity request as the first network request.
2. **Gene header in parallel.** `useGeneRecord(symbol)` fires on the same setup tick. The header card renders a skeleton until it resolves; cold target ≤ 100 ms.
3. **External hooks in parallel.** All five external hooks fire on the same setup tick, each owning its card's loading state. No `nextTick` deferral.
4. **Cancellation.** When the route param changes mid-flight (back/forward between genes), in-flight requests for the old key are aborted via the `useResource` `AbortController`.

### 4.3 Skeletons + layout

- `app/src/components/ui/SectionCard.vue` — wrapper taking `:loading`, `:empty`, `:error`, slots for header/body/actions. Skeleton stripes while loading; renders nothing when `empty && !loading` (today's "No constraint data available" empty card disappears entirely). Stable height during skeleton phase to avoid CLS.
- Header band: gene badge + name + chromosome location + resources + identifiers in one wrapping flex row at ≥ xl breakpoint; stacks at smaller breakpoints. Resources and identifiers share the same badge component for visual consistency. Gene name becomes the largest text element on the page (lifts hierarchy score).
- Below header: Associated Entities (full width, unchanged size — it is the data-dense block).
- Below entities: 3-up `<BRow>` at `lg+` for `Constraint | ClinVar | ModelOrgs` cards. At `<lg` it stacks. Each card hides itself when its hook resolves to "no data".
- `GenomicVisualizationTabs`: lazy-mount inactive tab panels (today they all mount on load). `<KeepAlive>` so switching tabs is instant after first activation.
- Remove duplicate fluid wrappers: replace `container-fluid > BContainer fluid > BRow > BCol` triple-nesting with a single `BContainer fluid > BRow > BCol` per section.
- Mobile: `TablesEntities` uses `BTable :stacked="md"` so it collapses to per-row labelled stacks below `md`.
- Sort-arrow noise: render only the active direction at idle; render both faintly only on hover/focus.
- Footer: collapse the logo strip to a smaller footer row to recover ~40 px of below-the-fold real estate.

### 4.4 EntityView gets the same treatment

- Mount `<EntityHistoryTable>` and the review list immediately on page mount.
- Header card uses the same skeleton + `<SectionCard>` pattern.
- Reuse the gene-info hook for the linked gene block.

### 4.5 Motion + feedback (low-cost)

- `<Transition mode="out-in">` between skeleton and content, 150 ms fade.
- Hover scale (1.02) + shadow on resource pills to confirm clickability.
- "Copied!" inline toast on identifier copy.
- Number reveal on KPI badges (count-up over 200 ms when a card resolves).

### 4.6 Accessibility

- Add `aria-live="polite"` to the entity table region.
- Add a skip-link "Jump to associated entities" rendered visibly on focus.
- Replace small-caps gray labels with theme tokens that satisfy AA contrast.
- Run `axe-core` against the page in the perf bench (Section 9) and fail if there are new violations vs. baseline.

## 5. Out of scope

- No backend changes (Plumber worker count, response shape, server-side caching).
- No SSR / Nuxt migration.
- No GraphQL or aggregated endpoint.
- No new design system. Bootstrap-Vue-Next + existing tokens stay.
- No new runtime dependency.
- The Ontology view, the Search view, and table list views are out of scope. They share *some* of the same patterns; the abstractions land here so those views can adopt later without a rewrite.

## 9. Verification harness

`app/tests/perf/genes-entities.bench.spec.ts` is a Playwright spec that:

1. Reads a small fixture list of two genes (GRIN2B, MECP2) and one entity (sysndd:400, MECP2).
2. Calls `make cache-clear` via `node:child_process` (skip on CI; local-only) before the cold pass.
3. For each gene/entity, hard-navigates to the page, polls the DOM for: skeletons present, header text, first entity row, all spinners gone, and uses `PerformanceObserver` for LCP and `web-vitals` for INP/CLS.
4. Writes timings to `.planning/perf/after-{date}.json` and a side-by-side diff vs. the baseline JSON in this PR.
5. Asserts the targets in Section 8.

The harness is **local-only** (per AGENTS.md, Playwright in this repo is local-only — there is no Playwright CI workflow). The acceptance numbers are committed in the perf JSON; PR review checks the diff.

## 11. Risks and mitigations

| Risk | Mitigation |
|------|------------|
| `useResource` SWR semantics drift from existing axios behaviour (interceptors, base URL, auth) | Build on top of the existing `apiClient` (`@/api/client`); inherit interceptors. |
| Stale cache surfaces wrong data after a privileged write (e.g., user updates an entity, reads stale view) | TTL is 60 s for our endpoints; mutations call `cacheStore.invalidate(prefix)` from the existing `useEntityMutations`. |
| Lazy tab content breaks cross-tab links (e.g., a click on a domain in Protein View jumps to Gene Structure) | Audit `GenomicVisualizationTabs` cross-tab refs in W2; if any panel publishes shared state, hoist that state into the parent before lazy-mounting. |
| Mobile stacked `BTable` regresses for power users on tablet | Breakpoint set at `md` (768 px) so iPad portrait still gets the table view. |
| Skeleton + content size mismatch causes CLS | Skeleton card minimum heights match the resolved content's typical heights; verified by the harness CLS gate. |
| Re-rate rubric is subjective | Lock the rubric text in this spec; re-score against the same screenshots at 1440 / 1024 / 375 px; require user-visible PR description to include the new screenshots side-by-side. |
