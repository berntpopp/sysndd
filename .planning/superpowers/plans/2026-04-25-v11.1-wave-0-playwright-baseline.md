# v11.1 Wave 0 — Playwright Baseline

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a Playwright regression baseline that runs against a real API+DB stack, with deterministic fixtures, before any refactor work begins. Wired into CI with screenshot artifacts.

**Architecture:** Playwright Test (Node) drives a real Chromium against a frontend served by `vite preview` and an API+DB stack brought up via a CI-only Docker Compose overlay (`docker-compose.playwright.yml`). Test users are seeded by a SQL fixture sourced only by the Playwright `make` target. Per-test isolation via unique names; between-run reset via `docker compose down -v`.

**Tech Stack:** Playwright 1.x, Docker Compose, MySQL, R/Plumber, Vite preview, GitHub Actions, ESLint 8 with `no-restricted-imports`.

**Worktree:** `sysndd-finish-hardening/` (the integration worktree).

**Commits:** ~20 commits, one per task.

// app/tests/e2e/fixtures/auth.ts
// Programmatic login that bypasses UI. Use this in non-auth tests so the
// auth flows remain the only place that exercises the login form.

import { test as base, type Page, type APIRequestContext } from '@playwright/test';
import { testUsers, type TestRole } from './test-users';

interface AuthFixtures {
  loggedInAs: (role: TestRole) => Promise<Page>;
}

async function fetchToken(request: APIRequestContext, role: TestRole): Promise<string> {
  const { username, password } = testUsers[role];
  const res = await request.post('/api/auth/authenticate', {
    data: { user_name: username, password },
  });
  if (!res.ok()) {
    throw new Error(`auth fixture: login failed for ${role}: ${res.status()} ${await res.text()}`);
  }
  // The endpoint returns the JWT as a JSON string (Plumber unboxes via
  // unboxedJSON serializer for this endpoint). Strip surrounding quotes
  // if present.
  const raw = await res.text();
  return raw.replace(/^"|"$/g, '');
}

export const test = base.extend<AuthFixtures>({
  loggedInAs: async ({ browser, request }, use) => {
    await use(async (role) => {
      const token = await fetchToken(request, role);
      const context = await browser.newContext();
      // Inject the token + a minimal user object into localStorage so the
      // app thinks the user is already logged in. Mirrors what useAuth
      // expects after a UI login.
      await context.addInitScript((args: { token: string; role: TestRole; users: typeof testUsers }) => {
        localStorage.setItem('token', args.token);
        const u = args.users[args.role];
        localStorage.setItem('user', JSON.stringify({ user_name: u.username, role: args.role }));
      }, { token, role, users: testUsers });
      const page = await context.newPage();
      return page;
    });
  },
});

export { expect } from '@playwright/test';
```

### Authoring a new spec

Use the auth fixture for non-auth tests:

```ts
import { test, expect } from './fixtures/auth';

test('something', async ({ loggedInAs }) => {
  const page = await loggedInAs('curator');
  await page.goto('/SomeRoute');
  // ...
});
```

Use `uniqueName('prefix')` from `fixtures/data.ts` for any server-side state created by the test. Tests must be order-independent — Playwright runs them in parallel by default.

### Screenshots

Tests write artifact screenshots to `app/tests/e2e/screenshots/<flow>__<step>__<viewport>.png`. The directory is gitignored; CI uploads it as a workflow artifact (retention 14 days).

### Selectors

The recommended workflow for selector discovery is `npx playwright codegen http://localhost:80/Path`. Prefer role + accessible-name selectors (`getByRole`, `getByLabel`) over CSS selectors.
