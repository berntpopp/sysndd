## Background: the migration pattern

Read once before starting any per-file task.

### Pattern

For every `this.axios.<verb>(...)` call site in the file:

1. Identify the URL string. It maps to a typed-client function in `app/src/api/`.
2. Add the import: `import { getFoo } from '@/api/genes'`. (Use the existing module's exports as ground truth — the typed clients exist in the merged W3 work.)
3. Replace:
   ```ts
   const response = await this.axios.get(apiUrl);
   const data = response.data;
   ```
   with:
   ```ts
   const data = await getFoo(id);
   ```
4. Drop the `apiUrl` const if no other call site uses it.
5. Drop value-imports of `axios` if no longer used: `import axios from 'axios'` → remove the line. Type-only imports (`import type { AxiosError } from 'axios'`) MAY remain.

### Mapping table for W4

These map known URL prefixes used in W4-owned files to their typed-client modules. Not exhaustive — refer to `app/src/api/*.ts` for the exact exports.

| URL prefix | Typed module | Common helpers |
|---|---|---|
| `/api/auth/*` | `@/api/auth` | `authenticate`, `signup` |
| `/api/user/*` | `@/api/user` | `getUser`, `updateUser`, `updatePassword`, `requestPasswordReset`, `changePasswordWithToken` |
| `/api/entity/*` | `@/api/entity` | `getEntity`, `createEntity`, `updateEntity`, `searchEntity`, `getEntityReview`, `getEntityPhenotypes`, `getEntityVariation`, `getEntityPublications` |
| `/api/review/*` | `@/api/review` | `getReview`, `approveReview`, `rejectReview` |
| `/api/status/*` | `@/api/status` | `getStatus`, `approveStatus`, `rejectStatus` |
| `/api/re_review/*` | `@/api/re_review` | `requestReReview` |
| `/api/list/*` | `@/api/list` | `getInheritanceList`, etc. |

If a URL doesn't have an obvious mapping, grep `app/src/api/` for the path:

```bash
grep -rn '/api/foo' app/src/api/
```

### Per-file task shape

Every per-file task uses the same five-step structure:

- Step 1: Inventory the raw axios usage in this file
- Step 2: Apply the migration pattern (above) for every call site
- Step 3: Run the existing Vitest spec for this file (if one exists) — green
- Step 4: Run the file-local grep — zero raw axios matches
- Step 5: Commit

The grep:

```bash
grep -nE "\bthis\.axios\.|^import axios( |,)|^import \* as axios " app/src/views/curate/Foo.vue
# Expected: zero output
```

Commit message template:

```
refactor(views/<area>/<File>): migrate to typed @/api/* clients

- Replaces N raw axios calls with typed helpers from @/api/<modules>.
- No behaviour change. Existing Vitest spec remains green.
```
