# v11.1 Wave 1a — W2: 401 Centralisation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `useAuth.handle401()` the single owner of logout cleanup. Remove the direct `localStorage.removeItem('token'/'user')` calls from the axios 401 interceptor; have it delegate to `useAuth().handle401()`. Add a Vitest spec covering the contract and a Playwright case in `auth.login.spec.ts` covering the user-facing behaviour.

**Architecture:** `useAuth` is the single source of truth for auth state. `axios.ts`'s response interceptor currently mutates `localStorage` directly on 401 — that creates a small window where reactive consumers (navbar, route guard) lag the persisted state. After this workstream, the interceptor calls a single `useAuth.handle401()` method that clears state in the order that downstream consumers expect, dispatches navigation to `/Login`, and is idempotent under concurrent rejections.

**Tech Stack:** Vue 3 Composition API, axios interceptors, Vitest, Playwright.

```ts
// Inside useAuth() — add handle401 if missing, replace if present.
//
// Single owner of 401 cleanup. Calling more than once is safe and a no-op
// after the first call. The order matters: clear localStorage first so any
// downstream `syncFromStorage()` reads see the cleared state, THEN clear
// reactive refs, THEN dispatch navigation. Reactive consumers (navbar,
// route guards) observe the ref clear immediately; persistence is already
// gone by the time any other component re-reads localStorage.
function handle401(): void {
  // Idempotency guard: if already cleared, just confirm navigation
  // without redoing the storage writes.
  const wasLoggedIn = token.value !== null || localStorage.getItem('token') !== null;

  localStorage.removeItem('token');
  localStorage.removeItem('user');

  token.value = null;
  user.value = null;

  if (wasLoggedIn) {
    // Either router.push (preferred) or a global hook for non-router callers
    // (e.g. tests). Use the router when available — the import is
    // top-level to avoid a circular import, but the resolution is lazy
    // because handle401 fires from interceptors that run after app init.
    try {
      const router = useRouter(); // or however the existing useAuth imports the router
      if (router && router.currentRoute.value.path !== '/Login') {
        router.push({ path: '/Login', query: { reason: 'session-expired' } });
      }
    } catch {
      // No router context (test or pre-init) — fall back to the hook.
      (globalThis as Record<string, unknown>).__authNavTarget = '/Login';
    }
  }
}

// Make sure handle401 is in the composable's return object
return {
  // ...existing members...
  handle401,
};
```

If the existing `useAuth` already has a logout function, refactor handle401 to delegate to it (same effect, no behaviour drift).

Pattern (adapt to the existing structure):

```ts
import { useAuth } from '@/composables/useAuth';

axios.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // Single owner of cleanup + navigation. Idempotent — concurrent 401s
      // converge to one cleared state.
      useAuth().handle401();

      // Tag the error so callers can detect "already handled" and skip
      // their own toast/log paths. Mirrors the prior contract.
      const handled = new Error('401 — session ended');
      (handled as Error & { __handled401: true }).__handled401 = true;
      return Promise.reject(handled);
    }
    return Promise.reject(error);
  },
);
```

Critically: do NOT leave any `localStorage.removeItem('token')` or `localStorage.removeItem('user')` calls in this file.

  test('tampered token redirects to login on next request and clears navbar state', async ({ loggedInAs }) => {
    const page = await loggedInAs('curator');
    await page.goto('/Entities');
    await expect(page.getByRole('navigation')).toContainText('pw_curator');

    // Tamper the persisted token so the next API call comes back 401
    await page.evaluate(() => {
      localStorage.setItem('token', 'tampered.invalid.token');
    });

    // Trigger an API call that will 401. Reloading or navigating to a
    // route that triggers a fetch is the simplest path.
    await page.goto('/User');

    // Expect: redirected to /Login (handle401 navigated us)
    await expect(page).toHaveURL(/\/Login/);

    // Expect: navbar reflects logged-out state
    await expect(page.getByRole('navigation')).not.toContainText('pw_curator');

    // Expect: token cleared from localStorage
    expect(await page.evaluate(() => localStorage.getItem('token'))).toBeNull();
    expect(await page.evaluate(() => localStorage.getItem('user'))).toBeNull();
  });
