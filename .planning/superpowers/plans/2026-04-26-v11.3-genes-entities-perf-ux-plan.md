# v11.3 Genes/Entities Perf + UX — Master Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the v11.3 perf + UX overhaul for `/Genes/:symbol` and `/Entities/:entity_id` — entities-first hydration, in-house SWR data layer, skeleton-driven progressive reveal, denser hide-when-empty layout, and a Playwright bench gate — in a single integration PR fed by four sequential workstreams.

**Architecture:** W1 lays the data layer (no view edits). W2 rewires `GeneView.vue`. W3 rewires `EntityView.vue`. W4 ships the perf bench, browser-axe runner, doc updates, version bump, and re-rates the UX rubric. Each wave runs in its own worktree; merges land on a shared integration branch.

**Tech Stack:** Vue 3 Composition API + TypeScript, Pinia, Bootstrap-Vue-Next, Vitest + MSW (unit/spec), Playwright + `@axe-core/playwright` (local-only bench), R/Plumber (API, unchanged), MySQL 8 (DB, unchanged), Docker Compose (`make dev`), GitHub Actions (CI).

**Spec:** `.planning/superpowers/specs/2026-04-26-v11.3-genes-entities-perf-ux-design.md`

**Branch:** `feature/v11.3-genes-entities-perf-ux` (created in M.1 below).

## Plan Layout

This master plan covers cross-wave setup, the merge gates between waves, and the final PR. Each workstream has its own executable plan file:

| Wave | Workstream | Plan file | Owner | Purpose |
|---|---|---|---|---|
| 1 | W1 | `2026-04-26-v11.3-wave-1-w1-data-layer.md` | W1 owner | `useResource` + Pinia `cacheStore` + 12 per-source hooks |
| 2 | W2 | `2026-04-26-v11.3-wave-2-w2-genes-page.md` | W2 owner | Rewire `GeneView.vue` — entities-first, `<SectionCard>`, 3-up grid, lazy tabs |
| 3 | W3 | `2026-04-26-v11.3-wave-3-w3-entities-page.md` | W3 owner | Rewire `EntityView.vue` — kill sequential awaits, six parallel hooks |
| 4 | W4 | `2026-04-26-v11.3-wave-4-w4-bench-closeout.md` | integration owner | Playwright bench, browser-axe, docs, version bump, rubric re-score, PR |

Workstream owners read **their plan file only**. The integration owner reads this master plan plus the W4 closeout plan.

Total: 1 master + 4 workstream plans = 5 files.

**Wave sequencing:** Unlike v11.2 (parallel waves), v11.3 is **sequential**. W2 depends on W1's hooks; W3 depends on W1's hooks plus the `<SectionCard>` introduced in W2; W4 measures all three and depends on a fully-converted view. Each wave merges before the next starts.
