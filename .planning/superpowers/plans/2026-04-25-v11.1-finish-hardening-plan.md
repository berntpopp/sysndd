# v11.1 Finish-Hardening — Master Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land all remaining P1+P2 items from the 2026-04-23 review (security headers, 401 centralisation, complete typed-API migration, decompose `Review.vue`) in a single PR fed by parallel worktrees, gated by a Playwright regression baseline that exists *before* any refactor begins.

**Architecture:** Hybrid wave model. Wave 0 establishes a Playwright regression baseline against a real API+DB stack (sequential commits to integration branch). Wave 1a runs three parallel worktrees (W1 security headers, W2 401 centralisation, W3 typed-client completion). Wave 1b runs four parallel worktrees that consume W3's contracts (W4 auth/curate, W5 analyses, W6 Review.vue decompose, W7 pages/tables/gene/misc). Wave 2 closes out (TS strict scope expansion, docs, PR).

**Tech Stack:** Vue 3 Options API + TypeScript (frontend), Vitest + MSW (unit/spec), Playwright (E2E + smoke + security headers), R/Plumber (API), MySQL 8 (DB), Docker Compose (CI stack), nginx (security headers), GitHub Actions (CI).

**Spec:** `.planning/superpowers/specs/2026-04-25-v11.1-finish-hardening-design.md`

**Branch:** `feature/v11.1-finish-hardening` (already created).
