- Clinical resources card-grid with 8 resources grouped by type (Curation, Disease/Phenotype, Gene Information, Model Organisms)
- Responsive grid layout (side-by-side on desktop, stacked on mobile/tablet)
- Simple spinner loading state
- Empty state "Not available" indicators for missing IDs

  <how-to-verify>
1. Start the development server: `cd /home/bernt-popp/development/sysndd/app && npm run dev`

2. Navigate to a gene page with complete data (e.g., http://localhost:5173/Genes/MECP2 or a gene symbol known to have most identifiers populated)

3. Verify hero section:
   - Gene symbol displayed as green 3D badge (large size)
   - Gene full name displayed prominently below/beside badge

4. Verify identifier card:
   - Card with "Identifiers" heading, subtle shadow, no visible border

   - Card with "Clinical Resources & Databases" heading

   - Available resources are clickable, open in new tab

   - Desktop (>992px): Two cards side by side
   - Resize browser to tablet width (~768px): Cards stack vertically
   - No horizontal scrolling at any width

7. Verify empty states:
   - Navigate to a gene with missing IDs (if known). Identifier rows show "Not available" in gray text instead of blank space.

8. Verify preserved functionality:
   - Associated entities table renders below the cards
   - Page title includes gene symbol
   - Navigate to a nonexistent gene — should redirect to PageNotFound

9. Verify loading state:
   - Hard refresh the page (Ctrl+Shift+R) — should see centered spinner briefly, then content appears all at once
