## v1 Requirements (All Complete)

### Gene Page Redesign (REDESIGN)

- [x] **REDESIGN-01**: Gene page displays hero section with symbol badge, gene name, chromosome location, and key identifiers prominently
- [x] **REDESIGN-02**: Gene identifiers are grouped into a card with copy-to-clipboard and external link buttons for each ID (Entrez, Ensembl, UniProt, UCSC, CCDS, STRING)
- [x] **REDESIGN-03**: Clinical resources are grouped into a visual card-grid with links to OMIM, ClinGen, SFARI, PanelApp, gene2phenotype, HGNC
- [x] **REDESIGN-04**: Model organisms section displays MGI and RGD links with availability indicators
- [x] **REDESIGN-05**: Loading skeleton placeholder renders while gene data loads (instead of spinner)
- [x] **REDESIGN-06**: Empty state indicators show "Not available" for missing IDs (instead of blank)
- [x] **REDESIGN-07**: Reusable IdentifierRow component handles copy-to-clipboard and external link for any database ID
- [x] **REDESIGN-08**: Reusable ResourceLink component renders card-style external resource links with icon and description
- [x] **REDESIGN-09**: Gene page uses responsive grid layout that works on desktop and tablet
- [x] **REDESIGN-10**: Gene.vue refactored to Composition API (script setup) using new sub-components

### Backend External API Proxy (PROXY)

- [x] **PROXY-01**: R/Plumber endpoint proxies gnomAD GraphQL API for gene constraint scores and ClinVar variants
- [x] **PROXY-02**: R/Plumber endpoint proxies UniProt REST API for protein domain and feature data
- [x] **PROXY-03**: R/Plumber endpoint proxies Ensembl REST API for gene structure (exons, transcripts)
- [x] **PROXY-04**: R/Plumber endpoint proxies AlphaFold API for 3D structure metadata and file URLs
- [x] **PROXY-05**: R/Plumber endpoint proxies MGI/JAX API for mouse phenotype data
- [x] **PROXY-06**: R/Plumber endpoint proxies RGD API for rat phenotype data
- [x] **PROXY-07**: Combined endpoint aggregates all external sources for a gene in a single request with error isolation
- [x] **PROXY-08**: Server-side disk caching (memoise + cachem) with configurable TTL per source
- [x] **PROXY-09**: httr2 retry logic with exponential backoff for all external API calls
- [x] **PROXY-10**: Rate limiting protection (req_throttle) to avoid exceeding external API limits
- [x] **PROXY-11**: External API endpoints are publicly accessible (no auth required) via AUTH_ALLOWLIST
- [x] **PROXY-12**: RFC 9457 error format for external API failures with source identification

### gnomAD Constraint Scores (GNOMAD)

- [x] **GNOMAD-01**: Gene page displays pLI score with visual gauge/indicator and interpretation
- [x] **GNOMAD-02**: Gene page displays LOEUF (oe_lof_upper) score with visual gauge and interpretation
- [x] **GNOMAD-03**: Gene page displays missense Z-score (mis_z) with visual indicator
- [x] **GNOMAD-04**: Constraint scores card shows observed vs expected counts for LoF, missense, and synonymous variants
- [x] **GNOMAD-05**: Constraint score thresholds use gnomAD v4 guidelines (pLI >= 0.9, LOEUF < 0.6 for constrained)

### ClinVar Variant Integration (CLINVAR)

- [x] **CLINVAR-01**: Gene page displays ClinVar variant summary card with counts by pathogenicity classification
- [x] **CLINVAR-02**: Summary uses ACMG color coding (pathogenic=red, likely pathogenic=orange, VUS=yellow, likely benign=light green, benign=green)
- [x] **CLINVAR-03**: ClinVar variants are fetched via gnomAD GraphQL API (clinvar_variants field)
- [x] **CLINVAR-04**: ClinVar variant positions are mapped onto the protein domain lollipop plot
- [x] **CLINVAR-05**: Lollipop plot supports filtering by pathogenicity classification (P/LP/VUS/LB/B)

### Protein Domain Visualization (PROTEIN)

- [x] **PROTEIN-01**: D3.js lollipop plot displays protein backbone with amino acid scale
- [x] **PROTEIN-02**: Protein domains from UniProt are rendered as colored rectangles on the backbone
- [x] **PROTEIN-03**: ClinVar variants are rendered as lollipop markers at their protein position
- [x] **PROTEIN-04**: Lollipop markers are colored by pathogenicity classification (ACMG colors)
- [x] **PROTEIN-05**: Hovering a variant marker shows tooltip with variant details (HGVS, classification, review status)
- [x] **PROTEIN-06**: Plot supports zoom and pan for exploring dense variant regions
- [x] **PROTEIN-07**: Domain legend shows color mapping for domain types
- [x] **PROTEIN-08**: Variant count indicator for positions with multiple overlapping variants

### Gene Structure Visualization (GENSTRUCT)

- [x] **GENSTRUCT-01**: Gene structure plot displays exons and introns from Ensembl canonical transcript
- [x] **GENSTRUCT-02**: Exons rendered as rectangles, introns as lines, with UTR regions distinguished
- [x] **GENSTRUCT-03**: Gene orientation (strand +/-) is indicated
- [x] **GENSTRUCT-04**: Chromosome coordinates displayed on axis

### 3D Protein Structure (STRUCT3D)

- [x] **STRUCT3D-01**: 3D viewer renders AlphaFold predicted structure with pLDDT confidence coloring
- [x] **STRUCT3D-02**: User can toggle between cartoon, surface, and ball+stick representations
- [x] **STRUCT3D-03**: User can highlight selected ClinVar variant on 3D structure with ACMG-colored marker
- [x] **STRUCT3D-04**: Variant panel lists available ClinVar variants with pathogenicity for selection
- [x] **STRUCT3D-05**: Viewer supports mouse pan, zoom, and rotate controls
- [x] **STRUCT3D-06**: Reset view button returns to default orientation
- [x] **STRUCT3D-07**: Graceful fallback message when AlphaFold structure is not available
- [x] **STRUCT3D-08**: Structure viewer properly cleans up WebGL resources on component unmount

### Model Organism Phenotypes (ORGANISM)

- [x] **ORGANISM-01**: Mouse phenotype card displays phenotype count from MGI with zygosity breakdown
- [x] **ORGANISM-02**: Rat phenotype card displays available phenotype data from RGD
- [x] **ORGANISM-03**: Phenotype data fetched via backend proxy (not direct frontend calls)
- [x] **ORGANISM-04**: Graceful empty state when no phenotype data available

### Frontend Composables (COMPOSE)

- [x] **COMPOSE-01**: useGeneExternalData composable fetches all external data via combined backend endpoint
- [x] **COMPOSE-02**: Composable exposes per-source loading states for independent card loading indicators
- [x] **COMPOSE-03**: Composable exposes per-source error states for graceful degradation
- [x] **COMPOSE-04**: useD3Lollipop composable manages D3 rendering lifecycle (create, update, destroy)
- [x] **COMPOSE-05**: use3DStructure composable wraps Mol*/NGL.js with markRaw() pattern for Vue 3 compatibility

### Accessibility (A11Y)

- [x] **A11Y-01**: All visualization cards have screen reader alternative text summarizing the data
- [x] **A11Y-02**: Constraint score gauges have aria-label with numeric values
- [x] **A11Y-03**: Copy-to-clipboard buttons have descriptive aria-labels
- [x] **A11Y-04**: 3D viewer has keyboard-accessible controls
- [x] **A11Y-05**: Color coding (ACMG) is supplemented with text labels (not color-only)
