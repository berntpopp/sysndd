## Background: the migration pattern

### Pattern (same as W4)

For every `this.axios.<verb>(...)`:

1. Identify the URL → typed-client mapping.
2. Add `import { ... } from '@/api/<module>'`.
3. Replace `const response = await this.axios.get(url); const data = response.data;` with `const data = await typedHelper(...);`.
4. Drop unused `apiUrl` consts.
5. Drop the value-import of `axios` if it becomes unused. Type-only imports may remain.

### Mapping table for W5

| URL prefix | Typed module |
|---|---|
| `/api/analysis/*` | `@/api/analysis` |
| `/api/comparisons/*` | `@/api/comparisons` |
| `/api/phenotype/*` | `@/api/phenotype` |
| `/api/publication/*` | `@/api/publication` |
| `/api/variant/*` | `@/api/variant` |
| `/api/list/*` | `@/api/list` |
| `/api/genes/*` | `@/api/genes` |

Refer to `app/src/api/<module>.ts` for the exact exports.

### Per-file task shape (same as W4)

5 steps: inventory → migrate → run spec (if any) → file-grep → commit.

The grep:

```bash
grep -nE "\bthis\.axios\.|^import axios( |,)|^import \* as axios " <file>
# Expected: zero output
```

Commit message template:

```
refactor(<area>/<File>): migrate to typed @/api/* clients
```
